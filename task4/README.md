# Task 4: Shopify Product Review App

A Shopify embedded app that lets customers submit reviews and displays them on the product page, with AI-powered summaries generated by the Claude API.

## Architecture

```
┌─────────────────────────┐      ┌──────────────────┐
│   Shopify Admin Panel   │      │  Storefront       │
│   (Embedded App)        │      │  (App Block)      │
│   /admin                │      │  /storefront/...  │
└──────────┬──────────────┘      └────────┬──────────┘
           │                              │
           ▼                              ▼
┌──────────────────────────────────────────────────────┐
│              FastAPI Backend (app.py)                 │
│                                                      │
│   POST /api/reviews      — Submit a review           │
│   GET  /api/reviews/:sku — Get reviews + AI summary  │
└──────────┬──────────────────────────────┬────────────┘
           │                              │
           ▼                              ▼
┌──────────────────┐          ┌────────────────────────┐
│   PostgreSQL     │          │   Claude API           │
│   (Task 1)       │          │   (AI Summaries)       │
└──────────────────┘          └────────────────────────┘
```

## Features
1. **Admin Panel** (`/admin`) — Merchants can view reviews, see AI summaries, and submit test reviews. In production, this would be embedded in the Shopify admin using App Bridge.
2. **Storefront Widget** (`/storefront/reviews/{sku}`) — Renders a review display and submission form. In production, this would be a Liquid App Block served via Shopify App Proxy.
3. **Review Submission** — Simple form: customer name, rating (1–5), and review text.
4. **AI Summary** — Uses Claude API to generate a one-sentence summary of all reviews for a product (e.g. "Customers love the texture but note the scent is strong").
5. **Database Storage** — Reviews are stored in the `product_reviews` table from the Task 1 schema, linked to product SKU.

## Setup

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Set API key
```bash
# Linux/Mac
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

# Windows PowerShell
$env:ANTHROPIC_API_KEY="sk-ant-your-key-here"
```
If no API key is set, the app falls back to a basic statistical summary.

### 3. (Optional) Configure PostgreSQL
```bash
export DB_HOST=localhost
export DB_NAME=teambeauty
export DB_USER=postgres
export DB_PASSWORD=postgres

# Apply schema from Task 1
psql -U postgres -d teambeauty -f ../task1/schema.sql
```
If PostgreSQL is unavailable, the app uses in-memory storage (data lost on restart).

### 4. Run the app
```bash
uvicorn app:app --reload --port 8001
```

### 5. Open the admin panel
Navigate to [http://localhost:8001/admin](http://localhost:8001/admin)

## Shopify Integration Notes

### How this would install on a real Shopify store:

1. **App Registration**: Register the app in the Shopify Partner Dashboard with the app URL pointing to your server.

2. **Admin Embed**: The `/admin` endpoint would be served inside the Shopify admin iframe using [Shopify App Bridge](https://shopify.dev/docs/api/app-bridge). The `shopify.app.toml` config file (included) defines the embed settings.

3. **Storefront Theme Extension**: The `extensions/product-reviews/` directory contains the App Block definition (`blocks/reviews.liquid`). This renders on the product page and fetches reviews via the Shopify App Proxy.

4. **App Proxy**: Configure in Shopify Partner Dashboard to route `/apps/reviews/*` to your server's `/storefront/reviews/*` endpoint.

You do **not** need to publish to the Shopify App Store. A working installation on a free Shopify Partner development store is sufficient.

## Design Notes
- **FastAPI over Node/Remix**: Chosen for consistency with Tasks 2 and 3. The Shopify CLI typically scaffolds a Remix app, but the core functionality (API + embedded UI) works equally well with any backend.
- **In-memory fallback**: Ensures the app works without PostgreSQL for demo purposes.
- **Claude Haiku**: Uses the smaller/faster Claude model for summaries since we only need one sentence.
